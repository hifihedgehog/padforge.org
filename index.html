What you can do with PadForge

Use any controller in any game

Plug a PS5 DualSense into a Steam game that only accepts Xbox pads. Plug a Logitech G29 wheel into a racing game that ignores wheels. Plug a Saitek HOTAS into a flight game that wants gamepads. The game sees whatever shape PadForge is set to.

Combine many devices into one virtual controller

Pedals plus a wheel plus a HOTAS throttle as one virtual stick. Left hand on a pad, right hand on a flight stick, both feeding the same output. Strongest / Combined / Average / Either / Both / Custom-formula combine modes. Cross-device chords. One row can read from any number of devices.

Layer your mappings with Shift keys

Each slot can carry extra mapping tables that activate while a button, chord, or axis is held. Five activation modes (Hold, Toggle, Sticky, Cycle, Custom jump-to). Per-layer color and emoji icon. A Win11-style flyout confirms which layer is engaged.

Aim with gyro at Steam Input parity

Reference frames (Local, Player, World), dual-threshold smoothing, real-world calibration, cross-device Aim Engage button. Per-(device, slot) tuning. Bind Gyro Pitch / Yaw / Roll as first-class sources in the Mappings table.

Drive triggers from games, audio, or constant force

Xbox Impulse Trigger passthrough so Forza, Gears, and Halo reach the trigger motors on the assigned physical pad. The same trigger data routes to DualSense as Adaptive Trigger Vibration. Plus audio-bass-driven trigger rumble and a constant-force override that resumes after game-driven force ends.

Keep what's special about your controller

Adaptive triggers in racing games. DualSense lightbar that pulses to game audio. Touchpad fingers forwarded. Strobe and Battery lightbar modes. Even if the game doesn't know what a DualSense is.

Play with whatever's in front of you

Phone in the room? Open a browser tab on it. The tab becomes a controller with sticks, D-pad, triggers, and rumble. Keyboard handy? Map WASD plus mouse to a virtual Xbox pad with sub-pixel mouse delta accumulation.

Run up to 16 controllers at once

Local co-op with mixed pad types. Two sim racers on two wheels. A flight stick plus throttle plus rudder pedals together as one virtual HOTAS. One combo press toggles every virtual controller on or off.

Map motion to emulators

Stream gyroscope and accelerometer to Cemu, Dolphin, Yuzu, and Ryujinx through the DSU / Cemuhook protocol on UDP port 26760. DualSense, DualShock 4, Switch Pro, and Switch 2 Pro sources all supported.

Make MIDI from a gamepad

Map sticks to Control Change. Map buttons to notes. Send velocity. Build a 16-channel MIDI controller from a gamepad and play music in any DAW.

Features

🎮

Any input, any output

Route any physical device to up to 16 stacked virtual controllers. Xbox, PlayStation, Extended (flight sticks, wheels, third-party gamepads), MIDI, or Keyboard+Mouse output. Mix types in one session.

🔎

3D and 2D visualization

Per-profile models with live input feedback. Xbox 360, Xbox One / Elite / Adaptive, Xbox Series, DualShock 4, and DualSense each have their own native art. Click any button in 3D to record its mapping.

⚙

Mapping that works the first time

Record by pressing a button, pick from a dropdown of every available input (including raw HID buttons past the standard 11), or run Map All for a one-pass setup. Copy bindings between controller types.

📈

Six deadzone shapes

Scaled Radial, Radial, Axial, Hybrid, Sloped Scaled Axial, Sloped Axial. Per-axis deadzone and anti-deadzone. Center calibration. Custom sensitivity curves with draggable points.

💫

Force feedback both ways

Full rumble passthrough. HID PID 1.0 force feedback for wheels and joysticks (constant, ramp, periodic, condition). Audio bass rumble that captures system audio and routes the bass to vibration motors.

🔌

Adaptive triggers and lightbar

Seven DualSense trigger modes including a one-click GameCube preset. Thirteen lightbar modes including audio-pulse and audio-band variants that react to game audio in real time.

⚡

Macros with action sequences

Combo triggers from up to 8 buttons, axes, or POV directions. Action sequences with key presses, mouse moves, scroll, system volume, app volume, lightbar overrides, and rumble overrides. Four fire modes.

👤

Per-app profiles

PadForge watches the foreground window and switches profiles when a matching app gains focus. A Win11-style flyout shows the active profile, initialization progress, and offline-controller warnings.

📡

1000 Hz polling

Sub-millisecond input jitter via high-resolution waitable timers. Bit-perfect axis passthrough at default settings. Built on SDL3 for broad device support.

🌐

Phone as a controller

Built-in web server. Open a browser tab on any phone or tablet over Wi-Fi. Touch buttons, dual analog sticks, 8-way D-pad, rumble. Xbox 360 or DualShock 4 layouts.

🔀

Motion server for emulators

Broadcast gyro and accelerometer over DSU/Cemuhook to Cemu, Dolphin, Yuzu, and Ryujinx. DualSense, DualShock 4, Switch Pro, and Switch 2 Pro sources.

🎵

MIDI output

Virtual MIDI endpoint. Axes send Control Change. Buttons send Note On / Off. Channel 1-16. PadForge creates its own system endpoint via Windows MIDI Services. No loopMIDI needed.

🚫

Hide physical controllers

HidHide driver-level hiding so games only see the virtual controllers. Per-device toggles auto-enable for gamepads. Optional cloak-between-launches for setups where Steam scans while PadForge is closed.

PadForge installs HIDMaestro on first run. HIDMaestro is the driver that creates virtual controllers. When you add a slot, HIDMaestro spins up a real HID device matching the controller shape you picked. The device disappears when you delete the slot.

Two more drivers are optional. PadForge offers to install each only when you need its feature:

  • HidHide — hide your physical controller so games only see the virtual one (when a game sees both at once)
  • Windows MIDI Services — only if you want the MIDI virtual controller type (Windows 11 24H2 or later)

Free means free. PadForge will never become paid, freemium, or Patreon early-access paywalled.
